# ADR-003：伏羲 v2.1 — 64易插件式前端服务架构方案

## 状态
提案中

## 上下文

### 当前状态
伏羲 v1.50 前端是一个**传统多页面 SPA（无框架）**：
- `index.html` 作为 Shell，通过 iframe 或 `<div>` 切换不同模块（登录/主面板/聊天/搜索/管理等）
- 路由通过 JS 手工控制 `window.location.hash`，无前端路由框架
- 服务页面（如 AI 工具、数据分析、文档工具、DXF 看图）**尚未独立前端实现**——后端 API 已就绪，前端缺少操作界面
- `vue3-migration/` 是 Vue 3 迁移工程，目前仅有路由框架（router/index.ts）和基础页面原型

### 核心约束
- **64易不是64个功能**，而是八卦（☰☷☳☴☵☲☶☱）的排列组合——任意两卦组合成一种服务模式
- 八卦核心框架（bagua/）负责基础设施调度，不应随外部服务增加而修改
- 每个外部服务是一个独立的"微应用窗口"，有自己的路由、状态、API 层
- 新服务接入 = 添加一个 `services/xxx/` 文件夹 + 4 个标准文件，无需碰核心八卦代码

### 后端已有结构梳理

#### 已接入微服务（4 个，25 个端点）

| 服务 | ID | 类型 | API Prefix | 端点数 | 端点清单 |
|------|----|------|-----------|--------|---------|
| AI智能工具 | `ai_tools` | ai | `/api/ai` | 6 | health, summarize, translate, keywords, entities, classify |
| 数据分析 | `data_analytics` | data | `/api/analytics` | 6 | health, stats(GET/POST), trends, report, storage, export |
| 文档工具 | `doc_tools` | utility | `/api/tools` | 8 | health, convert, merge, split, compress, compress-image, image-info, text-extract |
| DXF看图 | `dxf_viewer` | extension | `/api/dxf` | 5 | health, upload, files, view/{hash}, download/{hash} |

#### 核心 RAG（11 个模块）
chat / search / documents / wiki / graph / worldtree / feedback / auth / files / metadata / v2

#### 管理后台（7 个模块）
admin / dashboard / evaluation / evolution / feature-flags / metrics / health

#### 八卦基础设施（8 卦）
乾(知识检索) / 坤(记忆存储) / 震(主动推送) / 巽(数据管道) / 坎(安全风控) / 离(知识推理) / 艮(自修复) / 兑(对话交互)

#### 当前路由注册方式（server.py）


## 决策

采用**六层架构模型**，将前端从"单页 Shell + JS 切换"进化为"插件式微应用窗口操作系统"。

```typescript
/** 服务分类（对齐 business domain） */

/** 八卦符号（64易的基础） */
export type GuaSymbol = 'qian' | 'kun' | 'zhen' | 'xun' | 'kan' | 'li' | 'gen' | 'dui';

/** 卦组合 = 服务模式 */
export interface GuaCombination {
  primary: GuaSymbol;    // 主卦
  secondary: GuaSymbol;  // 辅卦
}

/** 单个 API 端点定义 */
export interface EndpointDef {
  method: 'GET' | 'POST' | 'PUT' | 'DELETE' | 'PATCH';
  path: string;              // 相对于 apiBase 的路径，如 /summarize
  description: string;
  requestSchema?: string;   // 请求体 JSON Schema（可选，用于自动表单生成）
  responseSchema?: string;  // 响应体 JSON Schema
}

/** 服务清单 = manifest.json 的类型定义 */
export interface ServiceManifest {
  // ── 标识 ──
  id: string;                // 全局唯一标识，kebab-case，如 'ai-tools'
  name: string;              // 显示名称，"AI智能工具"
  icon: string;              // 图标（emoji 或 iconify 图标名），如 '🤖' / 'mdi:robot'
  description: string;       // 一句话描述
  version: string;           // 语义化版本号

  // ── 分类与归属 ──
  category: ServiceCategory;
  guaAffinity?: GuaCombination;  // 归属的卦组合（可选）
  // 例：AI工具可挂载到 { primary: 'qian', secondary: 'li' }（乾×离 = 智能检索模式）

  route: string;             // 前端路由路径，如 '/services/ai-tools'
  apiBase: string;           // 后端 API 前缀，如 '/api/ai'
  endpoints: EndpointDef[];

  // ── 访问控制 ──
  requiredRole: 'user' | 'admin';
  featureFlag?: string;      // 功能开关 key，如 'service_ai_tools_enabled'

  // ── 窗口行为 ──
  windowMode: WindowMode;
  defaultSize?: {
    width: number;           // 默认宽度（px 或百分比字符串如 '60%'）
    height: number;          // 默认高度
  };
  defaultPosition?: {
    x: 'center' | number;
    y: 'center' | number;
  };
  resizable?: boolean;       // 是否可调整大小，默认 true
  minimizable?: boolean;     // 是否可最小化，默认 true

  // ── 依赖 ──
  dependencies?: string[];   // 依赖的其他 service id 列表
  singleton?: boolean;       // 是否只允许单实例窗口，默认 false（tab 模式通常 true）

  // ── 元数据 ──
  author?: string;
  tags?: string[];           // 标签，用于搜索
  changelog?: string;        // 变更说明
}
```

### 2.2 ServiceLoader 工作原理

```typescript
// _registry/ServiceLoader.ts
import type { ServiceManifest } from './types';

/**
 * 服务加载器 — 应用启动时执行
 * 
 * 执行流程：
 * 1. 扫描 services/ 目录下所有 manifest.json
 * 2. 校验 manifest 结构合法性（Zod schema）
 * 3. 按 feature flag 过滤 → 禁用的服务不注册
 * 4. 动态注册 Vue Router 路由
 * 5. 动态注册侧边栏/九宫格菜单项
 * 6. 向 ServiceRegistry 注册服务实例
 */
class ServiceLoader {
  private manifests: ServiceManifest[] = [];
  
  /** 使用 Vite 的 import.meta.glob 静态扫描 */
  async scan(): Promise<ServiceManifest[]> {
    // Vite 会在构建时解析 glob，生成静态 import map
    const manifestModules = import.meta.glob<{ default: ServiceManifest }>(
      '../*/manifest.json',
      { eager: false }
    );
    
    const manifests: ServiceManifest[] = [];
    const errors: { path: string; error: string }[] = [];
    
    for (const [path, loader] of Object.entries(manifestModules)) {
      try {
        const mod = await loader();
        const manifest = this.validate(mod.default);
        if (manifest) manifests.push(manifest);
      } catch (e) {
        errors.push({ path, error: String(e) });
      }
    }
    
    console.info(`[ServiceLoader] 发现 ${manifests.length} 个服务, ${errors.length} 个加载失败`);
    this.manifests = manifests;
    return manifests;
  }
  
  /** 获取启用的服务（检查 feature flag） */
  getEnabled(featureFlags: Record<string, boolean>): ServiceManifest[] {
    return this.manifests.filter(m => {
      if (!m.featureFlag) return true;
      return featureFlags[m.featureFlag] ?? true; // 未配置的 flag 默认启用
    });
  }
}

export const serviceLoader = new ServiceLoader();
```

### 2.3 ServiceRouter — 动态路由注册

```typescript
// _registry/ServiceRouter.ts
import type { Router } from 'vue-router';
import type { ServiceManifest } from './types';

/**
 * 将服务清单转化为 Vue Router 路由配置
 * 
 * 每个服务生成一条路由 + 可选的子路由
 */
export function registerServiceRoutes(
  router: Router,
  manifests: ServiceManifest[]
): void {
  for (const manifest of manifests) {
    // 只为主布局下的服务注册路由
    if (manifest.windowMode === 'modal') continue; // modal 不需要路由
    
    // 移除已注册的旧路由（支持热更新）
    const existingIndex = router.getRoutes().findIndex(r => r.path === manifest.route);
    if (existingIndex >= 0) {
      router.removeRoute(manifest.id);
    }
    
    router.addRoute('MainLayout', {
      path: manifest.route.replace(/^\//, ''),
      name: manifest.id,
      component: () => import(`../${manifest.id}/${toPascalCase(manifest.id)}Page.vue`),
      meta: {
        serviceId: manifest.id,
        serviceName: manifest.name,
        requiresAuth: true,
        requiresAdmin: manifest.requiredRole === 'admin',
        windowMode: manifest.windowMode,
      },
    });
  }
}

function toPascalCase(id: string): string {
  return id.split('-').map(w => w.charAt(0).toUpperCase() + w.slice(1)).join('');
}
```

### 3.3 guaAffinity 字段的设计意图

`guaAffinity` 不是强制字段——服务可以不绑定任何卦。但当服务绑定了卦组合后：
- 九宫格首页中，点击该卦格时，该服务会出现在卦格的下拉列表中
- 用户通过"卦组合筛选"（如选择"乾×离"）可以快速定位到相关服务
- 这本质上是一个**多维分类标签系统**，不增加耦合

```typescript
// 按卦组合查询服务
function getServicesByGua(
  primary: GuaSymbol,
  secondary: GuaSymbol
): ServiceManifest[] {
  return registry.getAll().filter(m => {
    if (!m.guaAffinity) return false;
    const { primary: p, secondary: s } = m.guaAffinity;
    // 卦组合是对称的：乾×离 ≡ 离×乾
    return (p === primary && s === secondary) || (p === secondary && s === primary);
  });
}
```

## 4. 窗口管理器 (WindowManager)

### 4.1 设计理念

借鉴操作系统窗口管理思想：每个微服务是一个独立的"窗口生命周期单元"。

```
核心概念对照：
  操作系统窗口  →  伏羲服务窗口
  ─────────────────────────────
  进程           →  ServiceInstance
  窗口           →  ServiceWindow（Vue 组件实例）
  任务栏         →  侧边栏/底部 Dock
  窗口管理器     →  WindowManager（Pinia Store）
  分屏           →  SplitLayout
  Alt+Tab       →  Ctrl+` 窗口切换
```

### 4.2 WindowManager Store

```typescript
// stores/windowManager.ts
import { defineStore } from 'pinia';
import type { ServiceManifest, WindowMode } from '@/types/service-manifest';

export interface ServiceWindow {
  id: string;              // 唯一窗口 ID = `${serviceId}_${instanceNumber}`
  serviceId: string;       // 服务 ID
  title: string;           // 窗口标题
  icon: string;            // 窗口图标
  state: WindowState;
  zIndex: number;          // 层叠顺序
  size: { width: number | string; height: number | string };
  position: { x: number; y: number };
  minimizedPosition?: { x: number; y: number };  // 最小化到的位置（动画用）
  openedAt: number;        // 打开时间戳
  data?: Record<string, any>;  // 窗口传递的上下文数据
}

export type WindowState = 'maximized' | 'normal' | 'minimized' | 'closed';

export const useWindowManager = defineStore('windowManager', {
  state: () => ({
    windows: [] as ServiceWindow[],
    activeWindowId: null as string | null,
    nextZIndex: 100,
    layout: 'cascade' as 'cascade' | 'tiled' | 'split-h' | 'split-v',
  }),
  
  getters: {
    /** 按 serviceId 分组的窗口列表（用于 Dock 显示） */
    windowsByService: (state) => {
      const map: Record<string, ServiceWindow[]> = {};
      for (const w of state.windows) {
        if (!map[w.serviceId]) map[w.serviceId] = [];
        map[w.serviceId].push(w);
      }
      return map;
    },
    
    /** 排序后的可见窗口（按 zIndex） */
    visibleWindows: (state) => 
      state.windows
        .filter(w => w.state !== 'minimized' && w.state !== 'closed')
        .sort((a, b) => a.zIndex - b.zIndex),
  },
  
  actions: {
    /** 打开服务窗口 */
    open(service: ServiceManifest, context?: Record<string, any>): string {
      // 检查 singleton
      if (service.singleton) {
        const existing = this.windows.find(
          w => w.serviceId === service.id && w.state !== 'closed'
        );
        if (existing) {
          this.focus(existing.id);
          return existing.id;
        }
      }
      
      const instanceNum = (this.windowsByService[service.id]?.length ?? 0) + 1;
      const window: ServiceWindow = {
        id: `${service.id}_${instanceNum}`,
        serviceId: service.id,
        title: service.name,
        icon: service.icon,
        state: (service.windowMode === 'fullscreen') ? 'maximized' : 'normal',
        zIndex: ++this.nextZIndex,
        size: {
          width: service.defaultSize?.width ?? 900,
          height: service.defaultSize?.height ?? 600,
        },
        position: this.calculateNextPosition(),
        openedAt: Date.now(),
        data: context,
      };
      
      this.windows.push(window);
      this.activeWindowId = window.id;
      return window.id;
    },
    
    /** 聚焦窗口 */
    focus(windowId: string) {
      const win = this.windows.find(w => w.id === windowId);
      if (!win) return;
      if (win.state === 'minimized') win.state = 'normal';
      win.zIndex = ++this.nextZIndex;
      this.activeWindowId = windowId;
    },
    
    /** 最小化 */
    minimize(windowId: string) {
      const win = this.windows.find(w => w.id === windowId);
      if (win) win.state = 'minimized';
      if (this.activeWindowId === windowId) {
        this.activeWindowId = null;
      }
    },
    
    /** 最大化/还原 */
    toggleMaximize(windowId: string) {
      const win = this.windows.find(w => w.id === windowId);
      if (!win) return;
      win.state = win.state === 'maximized' ? 'normal' : 'maximized';
    },
    
    /** 关闭 */
    close(windowId: string) {
      const win = this.windows.find(w => w.id === windowId);
      if (win) win.state = 'closed';
      // 延迟移除，用于关闭动画
      setTimeout(() => {
        this.windows = this.windows.filter(w => w.id !== windowId);
      }, 300);
      if (this.activeWindowId === windowId) {
        const remaining = this.windows.filter(w => w.state !== 'closed');
        this.activeWindowId = remaining.length > 0 ? remaining[remaining.length - 1].id : null;
      }
    },
    
    /** 拖拽移动 */
    move(windowId: string, x: number, y: number) {
      const win = this.windows.find(w => w.id === windowId);
      if (win) win.position = { x, y };
    },
    
    /** 调整大小 */
    resize(windowId: string, width: number, height: number) {
      const win = this.windows.find(w => w.id === windowId);
      if (win) win.size = { width, height };
    },
    
    /** 合并显示模式切换 */
    setLayout(layout: 'cascade' | 'tiled' | 'split-h' | 'split-v') {
      this.layout = layout;
      if (layout === 'tiled') this.arrangeTiled();
      if (layout === 'split-h' || layout === 'split-v') this.arrangeSplit(layout);
    },
    
    /** 分屏布局 */
    arrangeSplit(direction: 'split-h' | 'split-v') {
      const visible = this.visibleWindows;
      const count = visible.length;
      if (count === 0) return;
      
      const isHorizontal = direction === 'split-h';
      const viewportSize = isHorizontal
        ? window.innerWidth
        : window.innerHeight;
      const each = viewportSize / count;
      
      visible.forEach((w, i) => {
        w.size = isHorizontal
          ? { width: each - 8, height: '100%' }
          : { width: '100%', height: each - 8 };
        w.position = isHorizontal
          ? { x: i * each, y: 0 }
          : { x: 0, y: i * each };
        w.state = 'normal';
      });
    },
    
    /** 平铺布局 */
    arrangeTiled() {
      // 简单的网格平铺算法
      const visible = this.visibleWindows;
      const cols = Math.ceil(Math.sqrt(visible.length));
      const w = window.innerWidth / cols;
      
      visible.forEach((win, i) => {
        const row = Math.floor(i / cols);
        const col = i % cols;
        win.size = { width: w - 8, height: window.innerHeight / Math.ceil(visible.length / cols) - 8 };
        win.position = { x: col * w, y: row * win.size.height };
        win.state = 'normal';
      });
    },
    
    /** 计算下一个窗口的默认位置（层叠偏移） */
    calculateNextPosition() {
      const offset = (this.windows.length % 10) * 30;
      return { x: 100 + offset, y: 80 + offset };
    },
  },
});
```

### 4.5 ServiceEventBus — 窗口间通信

```typescript
// _registry/ServiceEventBus.ts

type EventHandler = (payload: any) => void;

/**
 * 服务事件总线 — 窗口间松耦合通信
 * 
 * 设计原则：
 * - 命名空间隔离：每个服务只能 emit 到自己的命名空间
 * - 全局事件使用 'global:*' 前缀（如 'global:theme-changed'）
 * - 不替代 Pinia Store（共享状态仍用 Store）
 * - 只用于通知类事件（"某件事发生了"），不用于请求-响应
 */
class ServiceEventBus {
  private handlers = new Map<string, Set<EventHandler>>();
  
  /** 发送事件 */
  emit(event: string, payload?: any, sourceServiceId?: string): void {
    const fullEvent = sourceServiceId
      ? `service:${sourceServiceId}:${event}`
      : event;
    
    // 通知具体事件的监听者
    this.notifyHandlers(fullEvent, payload);
    
    // 通知通配符监听者
    if (sourceServiceId) {
      this.notifyHandlers(`service:${sourceServiceId}:*`, payload);
    }
    this.notifyHandlers('*', { event: fullEvent, payload });
  }
  
  /** 监听事件 */
  on(event: string, handler: EventHandler): () => void {
    if (!this.handlers.has(event)) {
      this.handlers.set(event, new Set());
    }
    this.handlers.get(event)!.add(handler);
    
    // 返回取消监听的函数
    return () => {
      this.handlers.get(event)?.delete(handler);
    };
  }
  
  /** 一次性监听 */
  once(event: string, handler: EventHandler): void {
    const off = this.on(event, (payload) => {
      off();
      handler(payload);
    });
  }
  
  private notifyHandlers(event: string, payload?: any): void {
    this.handlers.get(event)?.forEach(h => h(payload));
  }
}

export const serviceEventBus = new ServiceEventBus();
```

## 6. 功能缺口分析 — "平台操作系统"定位

对照"平台操作系统"的定位，当前体系还缺少以下关键模块：

### 6.1 🔴 高优先级 — 阻塞性缺口

| 缺口 | 说明 | 为什么重要 | 建议实现 |
|------|------|----------|---------|
| **用户偏好与个性化** | 无用户级别的配置持久化（主题、语言、布局偏好） | OS 必须有多用户配置文件 | `UserPreferences` store + `/api/user/preferences` |
| **全局搜索入口** | 目前搜索是独立页面，不在各服务窗口内可用 | 任何 OS 都有 Cmd+K / Start Menu | 全局搜索栏（Ctrl+K 快捷键），可搜索服务、文档、命令 |
| **通知中心** | 无统一的通知机制 — 各服务各自为政 | OS 核心功能 | `/api/notifications` + WebSocket 推送 + 右上角通知铃铛 |
| **快捷键系统** | 只有基础键盘操作，无可配置快捷键 | 效率工具必备 | `ShortcutRegistry` — 注册/冲突检测/用户自定义 |
| **批处理/自动化** | 服务之间无法编排工作流 | 平台操作系统的核心价值 | 工作流引擎（如：上传 PDF → 提取文本 → AI 摘要 → 保存） |

### 6.2 🟡 中优先级 — 体验性缺口

| 缺口 | 说明 | 建议实现 |
|------|------|---------|
| **剪贴板/跨窗口传递** | 无法在服务窗口间拖拽或复制数据 | `CrossWindowClipboard` — 结构化数据传输（不是简单文本） |
| **窗口布局持久化** | 关闭浏览器后重新打开，所有窗口位置丢失 | `LayoutPersistence` — 保存到 localStorage/后端，恢复上次窗口布局 |
| **服务应用商店** | 新服务需要手动创建文件夹 | `ServiceStore` — 从远程 Registry 浏览/安装/卸载服务包 |
| **横屏/竖屏自适应** | 窗口管理器未考虑移动端/平板场景 | 响应式窗口策略 — 小屏自动最大化 |
| **多语言（i18n）** | 目前仅中文，无国际化框架 | Vue I18n 集成 + `manifest.json` 中的 i18n 键 |

### 6.3 🟢 低优先级 — 扩展性缺口

| 缺口 | 说明 | 建议实现 |
|------|------|---------|
| **插件沙箱** | 第三方服务可能不安全 | iframe 沙箱模式 + postMessage 通信 |
| **服务计费/配额** | 未来可能需要按用量计费 | `QuotaManager` — API 调用次数/存储用量追踪 |
| **A/B 测试框架** | 无法在服务级别做实验 | 服务级别的 feature flag 带百分比分流 |
| **离线模式** | 断网后服务不可用 | Service Worker + 离线缓存策略 |
| **服务依赖图可视化** | 无法直观看到服务间依赖 | D3.js 拓扑图展示 `manifest.dependencies` |
| **联邦搜索** | 无法跨服务统一搜索 | `/api/unified-search?q=xxx&services=ai-tools,doc-tools` |

### 6.4 建议新增的服务模块

基于"平台操作系统"定位，建议优先开发以下新服务：

1. **通知服务 (`notification-center`)**
   - API: `/api/notifications` — 消息列表、已读/未读、推送订阅、WebSocket 实时推送
   - 八卦归属: 震×兑 (推送 × 交互 = 智能通知模式)
   - 前端: 通知铃铛 + 侧边通知面板 + 通知偏好设置

2. **工作流引擎 (`workflow-engine`)**
   - API: `/api/workflows` — CRUD 工作流模板、触发执行、执行历史、当前状态
   - 八卦归属: 巽×离 (管道 × 推理 = 智能编排模式)
   - 前端: 可视化工流编辑器（拖拽节点连线）+ 执行日志

3. **用户中心 (`user-center`)**
   - API: `/api/user` — 个人资料、偏好设置、API Key 管理、使用统计
   - 八卦归属: 坤×坎 (存储 × 安全 = 安全档案模式)
   - 前端: 头像/设置面板/用量仪表板

4. **统一搜索 (`unified-search`)**
   - API: `/api/unified-search` — 跨服务全文搜索、搜索结果聚合
   - 八卦归属: 乾×坤 (检索 × 存储 = 全域搜索模式)
   - 前端: 全局搜索栏 (Ctrl+K) + 搜索结果分类展示

5. **服务商店 (`service-store`)**
   - API: `/api/service-store` — 浏览/搜索/安装/卸载服务包
   - 八卦归属: 巽×乾 (管道 × 检索 = 能力发现模式)
   - 前端: 卡片式服务浏览 + 一键安装

### 7.1 服务路由自动注册（当前最大痛点）

```python
# src/services/_auto_discovery.py — 替代 server.py 中的手写 include_router

import importlib
import json
from pathlib import Path
from fastapi import FastAPI

SERVICES_DIR = Path(__file__).parent  # src/services/

def discover_and_register_services(app: FastAPI):
    """自动扫描微服务目录并注册路由"""
    registered = []
    
    for service_dir in SERVICES_DIR.iterdir():
        if not service_dir.is_dir() or service_dir.name.startswith('_'):
            continue
        
        manifest_path = service_dir / 'service.json'
        if not manifest_path.exists():
            continue
        
        manifest = json.loads(manifest_path.read_text(encoding='utf-8'))
        service_id = manifest.get('id', service_dir.name)
        
        # 尝试导入 routes 模块
        routes_mod = None
        for mod_name in ('routes', 'api'):
            try:
                routes_mod = importlib.import_module(
                    f'src.services.{service_dir.name}.{mod_name}'
                )
                break
            except (ImportError, ModuleNotFoundError):
                continue
        
        if routes_mod and hasattr(routes_mod, 'router'):
            prefix = manifest.get('api_prefix', '')
            app.include_router(routes_mod.router, prefix=prefix)
            registered.append(service_id)
    
    print(f'[AutoDiscovery] 注册 {len(registered)} 个服务: {registered}')
    return registered
```

在 server.py 中只需一行：
```python
from src.services._auto_discovery import discover_and_register_services
discover_and_register_services(app)
```

```python
    manifests = []
    import glob
    for service_dir in glob.glob("src/services/*/service.json"):
```

## 9. 权衡分析

### 获得的
- ✅ **可扩展性**：新服务接入不需要动八卦核心代码，只需 4 个标准文件
- ✅ **运行时可管理**：功能开关可以在不重启的情况下启用/禁用服务
- ✅ **多窗口工作流**：用户可同时使用 AI 工具 + 数据分析 + 文档转换，窗口独立管理
- ✅ **松耦合**：服务间通过事件总线通信，不产生硬依赖
- ✅ **渐进迁移**：Vue 3 迁移 + 插件架构可以逐步推进，不需要一次性重写全部前端
- ✅ **64易语义对齐**：八卦×八卦=服务模式的映射让系统具有文化内涵，而不只是技术实现

### 付出的
- ❌ **复杂度增加**：WindowManager + ServiceLoader + ServiceEventBus 共约 500 行核心代码需要维护
- ❌ **学习成本**：新开发者需要理解"微应用窗口"概念和 manifest.json 规范
- ❌ **性能考虑**：多个 Canvas/DOM 窗口同时打开时的内存管理（KeepAlive max=10 只是缓解）
- ❌ **当前前端仍是 Vanilla JS**：完整的插件架构需要 Vue 3 迁移完成后才能落地。在此之前可以用简化版（JSON 配置驱动 + 动态 script 加载）

### 短期折中方案（Vue 3 迁移完成前）
1. 在 `frontend/js/services.js` 中实现简化版 ServiceLoader（读取 JSON + 动态创建 DOM 面板）
2. 窗口管理可以用 `<div>` + CSS Grid/Flexbox 模拟，暂不实现拖拽
3. 事件总线用 `CustomEvent` + `window.dispatchEvent` 替代
4. `manifest.json` 格式与最终 Vue 3 版本保持一致，确保平滑迁移
