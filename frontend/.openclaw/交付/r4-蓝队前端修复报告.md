## 详细修复内容

### 1. SRI 校验（index.html）
- d3.v7.min.js: `sha384-CjloA8y00+1SDAUkjs099PVfnY2KmDC2BZnws9kh8D/lX1s46w6EPhpXdqMfjK6i`
- chart.umd.min.js: `sha384-vsrfeLOOY6KuIYKDlmVH5UiBmgIdB1oEf7p01YgWHuqmOHfZr374+odEv96n9tNC`
- 添加 `crossorigin="anonymous"` 属性

### 2. Token 加密存储
- **api-client.js**: `setAuthSync()` 改用 `_fallbackEncode()` (XOR+base64 混淆)
- **login.html**: 新增 `_encodeToken()` 函数（与 api-client.js fallback 一致），登录成功后加密存储 token
- **login.html**: 自动登录检查增加解码逻辑，兼容加密存储的 token

### 3. CSP 统一
- index.html: 添加 `base-uri 'self'`、`form-action 'self'`、`frame-ancestors 'none'`
- login.html: 同步添加上述 CSP 指令
- 移除 `unsafe-eval`（当前 CSP 中未包含，已确认）

### 4. SSE 自动重连
- 新增 `_SSE_MAX_RETRIES = 3` 和 `_SSE_BASE_DELAY = 1000` 常量
- 重连循环：网络错误/超时/5xx → 指数退避重试
- 不重试：认证错误(401/403)、用户主动中止(AbortError)、频率限制(429)
- 重试时显示提示："连接中断，正在重连 (1/3)…"
- 成功后 `return` 退出；失败后 `break` 退出循环并显示错误

### 5. 分页机制
- **Wiki 树**: `_WIKI_PAGE_SIZE = 50`，每个分类最多显示 50 条，超出显示"加载更多"按钮
- **文件列表**: `_FILE_PAGE_SIZE = 20`，首次显示 20 条，点击"加载更多"追加下一批
- 切换分类/搜索时自动重置分页偏移量

### 6. 移动端响应式
- **1024px**: Wiki 侧栏收窄至 200px，文件网格 minmax(200px)，图谱面板 220px
- **768px**: 侧边栏固定定位+全屏高度，Wiki树限高 40vh，对话区域 padding 收窄，登录卡片 90% 宽
- **480px**: 统计卡片单列，文件卡片单列，消息气泡 95% 宽，品牌文字隐藏，搜索框隐藏
