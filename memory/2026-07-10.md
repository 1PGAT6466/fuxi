## 发现的关键漏洞

### P0（严重）
1. **租户注入漏洞**：注册端点允许用户自行指定 tenant_id，没有验证是否存在
2. **Token 黑名单内存存储**：服务重启后黑名单清空，旧 Token 可复活

### P1（中危）
3. **CSRF Token 无效**：前端发送 CSRF Token，但后端没有验证
4. **错误信息泄露**：某些端点直接返回 detail=str(e)，可能泄露内部路径

### P2（低危）
5. 弱密钥黑名单不完整
6. RBAC 单例无线程锁
7. Markdown 渲染可能存在 XSS

## 安全亮点
- JWT 签名验证机制完善（HS256 + algorithms 限制）
- Token 版本号检查有效
- 前端 Token 加密存储（AES-GCM）
- Prompt Injection 防御三层架构
- 速率限制和账号锁定机制

## 检测方法
- 代码审计：逐函数分析安全逻辑
- 静态分析：检查输入验证、权限检查、数据隔离
- 逻辑推演：构造攻击链验证漏洞可利用性
