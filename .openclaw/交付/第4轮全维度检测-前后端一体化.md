#### M-2: MMR 去重算法 O(n²×k) 字符级交集

**文件**: `src/services/results_postprocess.py:50`

```python
max_overlap = max(
    (len(set(text_i) & set(st)) / max(len(set(text_i) | set(st)), 1)
     for st in selected_texts), default=0
)
```

问题：
- `set(text_i)` 将整个文本拆成**字符集合**，而非词级/句级语义去重
- 每次外层循环都要遍历所有已选文本，复杂度 O(n²×L)（L = 文本长度）
- 字符级 Jaccard 相似度对中文语义去重效果差（"知识管理" 和 "管理知识" 字符集相同）

**修复建议**:
```python
# 方案 A: 使用前 N 字符的哈希作为轻量去重
def _text_signature(text: str, n: int = 100) -> str:
    return text[:n].strip()

# 方案 B: 使用 n-gram Jaccard（2-gram 或 3-gram）
from functools import lru_cache
@lru_cache(maxsize=256)
def _ngrams(text: str, n: int = 2) -> frozenset:
    return frozenset(text[i:i+n] for i in range(len(text) - n + 1))
```

#### M-3: CSRF Token 未实现

**文件**: `src/api/auth.py`、`src/middleware.py`

当前系统使用 JWT Bearer Token 认证，无 CSRF 保护。虽然 JWT 本身不依赖 Cookie（不易受 CSRF 攻击），但存在以下风险：

- 如果前端将 Token 存储在 `localStorage`，XSS 攻击可窃取 Token
- 如果未来切换到 Cookie 存储 Token，则需要 CSRF 保护

**当前状态**: 风险可控（JWT + Authorization header 模式），但建议：
1. 确保前端 Token 仅存 `localStorage`（非 Cookie）
2. 添加 `SameSite=Strict` Cookie 策略（如果使用 Cookie）
3. 考虑实现 `X-CSRF-Token` 双重提交验证（低优先级）
