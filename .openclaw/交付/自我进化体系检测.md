### 链路验证
```
用户反馈 → feedback_store（MD5 去重 + TTL 缓存）
  → learn_buffer 积累（阈值 20 条）
  → learner（术语权重 ±0.1/0.05，个性化 boost）
  → evolver（regex 实体发现 + 共现关系推理）
  → knowledge_graph.json 增量更新（原子写入）
  → lifecycle（事件记录 → 触发检查 → 候选获取）
```

### 日/周/月循环
- **日循环**：`dream_cycle.py` 实现 digest/enrich/consolidate/gap_scan/health_report 五个阶段，设计为每日 02:00 cron 触发
- **周循环**：`eval_updater.py` 每 7 天从搜索日志提取高频查询，自动更新评测集
- **月循环**：未见显式月度循环实现（可通过 cron 手动配置）

### 评估
- ✅ 反馈收集：支持 like/dislike/correct/copy/click 五种动作
- ✅ 去重机制：MD5 签名 + TTLCache（300 秒窗口）
- ✅ 批量学习：buffer 满 20 条自动触发，失败重试最多 3 次
- ✅ Dream Cycle：零 LLM 设计（digest/enrich/consolidate 全基于向量相似度 + 正则）
- 🟡 月循环：未内置，需通过外部 cron 配置

### 链路验证

- ✅ 置信度三级分类（high/medium/low）
- ✅ JSONL 持久化存储
- 🟡 自动添加（auto_add）逻辑未在 knowledge_lifecycle 中直接实现，需由上层 evolver 消费

## 修复建议

### 优先级 P1（建议尽快处理）
1. **A/B 测试支持**：当前 Feature Flag 仅支持布尔开关。如需灰度发布/实验，建议扩展 `feature_flags.py` 增加 `experiment_config`（流量比例、用户分组、指标统计）
2. **月度循环**：Dream Cycle 已实现日循环，eval_updater 实现周循环，但无内置月度归档/总结逻辑。建议在 `dream_cycle.py` 增加月度汇总任务

### 优先级 P2（优化项）
3. **实体消歧**：`evolver.py` 的 `discover_entities()` 使用 regex 匹配，同名不同义实体无法区分。建议引入上下文窗口消歧或 LLM 辅助
4. **评测 Dashboard 前端**：后端 API 已完备（`/api/evaluation/*`），但未发现独立的前端评测可视化组件
5. **Wiki 自动生成**：当前 Wiki 页面需手动创建，建议增加从高频 chunk 聚合自动生成 Wiki 页面的管线
6. **knowledge_lifecycle auto_add**：置信度 ≥0.9 的候选知识标记为 auto_add，但实际自动添加逻辑在 evolver 中，建议在 lifecycle 层增加直接写入能力

### 优先级 P3（锦上添花）
7. **图谱可视化**：`/api/graph` 返回 JSON 数据，建议增加前端力导向图可视化
8. **评测指标扩展**：可增加 Answer Correctness、Context Recall 等 RAGAS 标准指标
