## 架构总结

伏羲 v1.44 的自我进化体系采用 **八卦框架（Bagua）** 架构，核心为 **第九宫·中宫（EvolutionGua）**：

```
                    ┌─────────────────────────────┐
                    │     EvolutionGua (中宫 ⊙)    │
                    │   统一执行入口 + 降级矩阵     │
                    └──────────┬──────────────────┘
                               │ execute({"action": ...})
          ┌────────────────────┼────────────────────┐
          │                    │                    │
    ┌─────▼─────┐    ┌────────▼────────┐    ┌─────▼─────┐
    │ FeedbackLoop│    │ EvolutionLearner│    │EvolutionEvolver│
    │  反馈记录   │    │  术语权重调整   │    │  实体发现    │
    │  去重+批量  │    │  个性化 boost  │    │  关系推理    │
    └─────┬─────┘    └────────┬────────┘    └─────┬─────┘
          │                    │                    │
    ┌─────▼─────┐    ┌────────▼────────┐    ┌─────▼─────┐
    │feedback_store│  │  services/      │    │services/   │
    │  (services) │   │  learner.py     │    │evolver.py  │
    └───────────┘    └─────────────────┘    └───────────┘

                    ┌─────────────────────────────┐
                    │    EvolutionLifecycle         │
                    │  知识事件 → 触发检查 → 候选   │
                    └──────────┬──────────────────┘
                               │
                    ┌──────────▼──────────────────┐
                    │  knowledge_lifecycle.py       │
                    │  JSONL 事件日志 + 置信度分类   │
                    └─────────────────────────────┘
```

**数据流**：
```
用户反馈 → feedback_store → Learner(术语权重) → Evolver(图谱更新)
                                                      ↓
知识生命周期事件 → KnowledgeLifecycle → 触发条件检查 → 候选知识
                                                      ↓
DreamCycle(夜间) → digest/enrich/consolidate/gap_scan → 日报
```

**Feature Flag 控制**：25 个开关覆盖全模块，10 秒自动刷新，WebSocket 实时广播。
