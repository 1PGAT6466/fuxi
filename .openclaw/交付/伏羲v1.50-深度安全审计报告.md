## 🔐 1. 认证安全审计

### 1.1 JWT 配置 ✅ 合格


**优点:**
- ✅ 强制要求环境变量设置 JWT_SECRET
- ✅ 禁止使用默认密钥
- ✅ Token 有效期 24 小时合理
- ✅ 使用 HS256 算法

**问题:**
- ⚠️ **Token 有效期过长:** 24 小时对于高安全场景过长
- ⚠️ **无 Token 刷新机制:** 用户需重新登录
- ⚠️ **无 Token 黑名单:** 无法主动撤销已发放的 Token

**风险等级:** 中

### 1.2 密码加密 ✅ 合格

**代码位置:** `src/api/auth_routes.py`

```python
def _hash_password(password: str) -> str:
    return bcrypt.hashpw(password.encode(), bcrypt.gensalt()).decode()

def _verify_password(password: str, stored: str) -> bool:
    if stored.startswith("$2b$") or stored.startswith("$2a$"):
        return bcrypt.checkpw(password.encode(), stored.encode())
    # v1.50: 旧格式密码 — 验证失败，强制重置
    return False
```

**优点:**
- ✅ 使用 bcrypt 加密（业界标准）
- ✅ v1.50 移除了不安全的 sha256+salt 旧格式
- ✅ 强制旧密码重置
- ✅ 密码长度限制 6-50 字符

**问题:**
- ⚠️ **密码复杂度未强制:** 未要求包含大小写字母、数字、特殊字符
- ⚠️ **无密码历史检查:** 可重复使用旧密码

**风险等级:** 低

```python
SENSITIVE_PATTERNS: List[re.Pattern] = [

    re.compile(r"(secret|token|api[_-]?key)\s*[:=]\s*\S+", re.I),
```

## 📝 审计结论
