## 三、前后端依赖一致性

### 3.1 API 客户端版本

| 维度 | 后端 | 前端 | 一致性 |
|------|------|------|--------|
| HTTP 框架 | FastAPI (Python) | 原生 fetch / axios | ✅ 异构正常 |
| WebSocket | - | 原生 WebSocket | ✅ 无版本依赖 |

### 3.2 协议版本

| 协议 | 后端版本 | 前端版本 | 状态 |
|------|----------|----------|------|
| HTTP | HTTP/1.1 (uvicorn) | HTTP/1.1 | ✅ 一致 |
| API 路径 | `/api/*` | `/api/*` (proxy) | ✅ 一致 |

### 3.3 共同依赖

| 依赖 | 后端 | 前端 | 一致性 |
|------|------|------|--------|
| DOMPurify | Python: 无 | JS: dompurify.min.js | 🟡 前端有，后端无 |
| Markdown | mistune>=3.1.0 | marked.min.js | 🟡 实现不同 |

#### 🟡 DOMPurify 使用差异

- **前端**：使用 DOMPurify 进行 XSS 防护（聊天、知识库内容渲染）
- **后端**：未引入 HTML 净化库

**风险**：如果后端直接返回未净化的 HTML 内容，存在存储型 XSS 风险。  
**建议**：后端 API 返回 HTML 内容时，应使用 `bleach` 或 `nh3` 进行净化。
