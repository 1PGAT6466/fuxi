## 五、API 响应格式一致性（核心问题）

### 5.1 统一格式定义

系统定义了统一响应格式（`src/api/response.py`）：
```json
// 成功
{"status": "success", "message": "ok", "data": {...}}
// 错误
{"status": "error", "message": "错误描述", "status_code": 400}
```

### 5.4 429 速率限制格式不统一

**严重程度：🟡 MEDIUM**

slowapi 速率限制返回：
```json
{"detail": "请求过于频繁，请稍后再试", "retry_after_seconds": 300}
```

应统一为：
```json
{"status": "error", "message": "请求过于频繁，请稍后再试", "status_code": 429}
```

**根因**：`src/middleware.py` 中 slowapi 使用默认 `_rate_limit_exceeded_handler`，未覆盖为统一格式。

## 六、500 错误诊断（已修复）

### 问题

测试过程中发现 `/api/metadata`、`/api/dashboard`、`/api/evolution/overview` 返回 500：
```json
{"error":"Internal server error","detail":"unexpected indent (vector_store.py, line 426)"}
```

### 根因

**stale .pyc 缓存文件**。`src/db/__pycache__/vector_store.cpython-311.pyc` 是旧版本的编译缓存，与当前源码不一致。

### 修复

删除 stale .pyc 文件后，所有端点恢复正常：
```powershell
Remove-Item "src\db\__pycache__\vector_store.cpython-311.pyc"
```

### 建议

在 `start_server.py` 或部署脚本中添加 .pyc 清理步骤：
```python
import shutil, pathlib
for pyc in pathlib.Path("src").rglob("__pycache__"):
    shutil.rmtree(pyc, ignore_errors=True)
```

## 九、问题汇总与优先级

### 🟡 MEDIUM（建议修复）

1. **15 个端点响应格式不统一**
   - 搜索、文档、Wiki、图谱、聊天等核心业务端点未使用统一格式
   - **影响**：前端需要为每个端点写不同的解析逻辑

2. **429 速率限制格式不统一**
   - slowapi 使用默认 handler，返回 `{detail: "..."}`
   - **修复**：覆盖 `_rate_limit_exceeded_handler`

3. **stale .pyc 缓存导致 500 错误**
   - 删除缓存后恢复，但需要预防措施
   - **修复**：启动时清理 __pycache__

### 🟢 LOW（可选优化）

4. **403 错误消息不准确**
   - 返回 `"请求参数错误"` 而非 `"权限不足"`

5. **ChromaDB 权限警告**
   - 数据库文件权限过宽（0o666），建议收紧

6. **API 路径文档**
   - 部分端点路径与直觉不符（如 `/api/worldtree/stats` 而非 `/api/worldtree`）
