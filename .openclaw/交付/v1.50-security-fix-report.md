## 详细修复内容

### 1. API 端点权限控制缺失

**问题**: `/api/feature-flags`, `/api/eval/run`, `/api/system/stats` 无权限检查，任何登录用户均可访问。

**修复**: 
- 在 `server.py` 中添加 `_require_admin(request)` 辅助函数
- 为 `/api/feature-flags` (GET/PUT), `/api/eval/run` (POST), `/api/system/stats` (GET) 添加管理员权限检查
- 非管理员用户访问时返回 `403 Forbidden`

**影响范围**: 仅影响非管理员用户的访问权限

---

### 2. 数据隔离实现

**问题**: 所有用户可看到全部文档，无数据隔离。

**修复**:
- `chunks` 表新增 `uploaded_by` 字段（自动迁移已有数据库）
- `Chunk` 数据模型新增 `uploaded_by` 属性
- `MemoryStore.add()` 和 `add_batch()` 支持 `uploaded_by` 写入
- `documents` API 按用户过滤：非管理员只能看到自己上传的文档
- 上传接口将当前用户写入 `uploaded_by`

**影响范围**: 
- 管理员可看到所有文档
- 普通用户只能看到自己上传的文档
- 已有文档的 `uploaded_by` 为空，对所有用户可见（兼容）

---

### 3. CSRF 防护实现

**问题**: 缺少 CSRF Token 防护，存在跨站请求伪造风险。

**修复**:
- 新增 CSRF 中间件：所有 `/api/` 写操作（POST/PUT/DELETE）必须携带 `X-CSRF-Token`
- 新增 `/api/csrf-token` 端点：登录后可获取 CSRF Token（1小时有效）
- CORS 配置增加 `X-CSRF-Token` header 支持
- 白名单: 登录/注册/健康检查/MCP 协议免检

**使用方式**:
```
1. 登录获取 token
2. GET /api/csrf-token 获取 csrf_token
3. 后续写请求 Header 携带 X-CSRF-Token: <csrf_token>
```

---

### 4. Token 有效期优化

**问题**: JWT Token 有效期 24 小时，过长。

**修复**:
- Access Token 有效期: 24h → **2小时**
- 新增 Refresh Token 机制（7天有效）
- 登录接口同时返回 `token` 和 `refresh_token`
- 新增 `/api/auth/refresh` 端点，支持用 refresh token 换取新 access token

**影响范围**: 前端需适配 token 刷新逻辑，避免用户每 2 小时重新登录

---

### 5. API 文档禁用

**问题**: 生产环境暴露 `/docs` 和 `/redoc`，泄露 API 结构。

**修复**:
- 通过环境变量 `FUXI_ENV` 控制
- 当 `FUXI_ENV=production` 时，`/docs` 和 `/redoc` 返回 404
- 开发环境（默认）仍可访问

**配置**: 在 `.env` 中设置 `FUXI_ENV=production`

---

### 6. 文件上传白名单

**问题**: 允许上传 `.py`, `.js`, `.sh`, `.bat` 等可执行文件。

**修复**:
- 从 `ALLOWED_EXTENSIONS` 中移除: `.py`, `.js`, `.ts`, `.java`, `.c`, `.cpp`, `.h`, `.sh`, `.bat`, `.ps1`
- 上传接口新增双重黑名单检查（防止白名单被误修改）
- 禁止的扩展名: `.py`, `.js`, `.sh`, `.bat`, `.ps1`, `.exe`, `.dll`, `.cmd`, `.vbs`, `.msi`, `.jar`, `.war`, `.php`, `.asp`, `.aspx`

**影响范围**: 需要上传代码文件的用户将被拒绝，建议通过 Git 等其他方式管理代码

---

### 7. XSS 防护增强

**问题**: 缺少 XSS 过滤规则。

**修复**:
- 在 `security.py` 中新增 `XSS_PATTERNS` 列表，覆盖:
  - `<script>`, `<iframe>`, `<object>`, `<embed>`, `<link>` 标签
  - `javascript:`, `vbscript:` 协议
  - `on*` 事件处理器（onclick, onload 等）
  - `expression()`, `url()` CSS 表达式
  - 内联 `style`, `formaction`, `data` 属性
- 新增 `sanitize_xss(text)` 函数，自动移除危险内容并转义 HTML 实体

---

### 9. 日志脱敏强制使用

**问题**: 日志中可能泄露敏感信息。

**修复**:
- `auth.py` 中的 Token 验证失败日志使用 `sanitize_for_log()` 脱敏
- `config.py` 中已有 `sanitize_for_log()` 函数和 `SENSITIVE_PATTERNS` 配置
- 脱敏规则覆盖: password, secret, token, api_key, 身份证号, 手机号

## 部署注意事项

1. **Token 刷新**: 前端需适配 `/api/auth/refresh` 接口，避免用户频繁重新登录
2. **CSRF Token**: 前端写操作需先获取 CSRF Token 并携带在 Header 中
3. **生产环境**: 设置 `FUXI_ENV=production` 禁用 API 文档
4. **数据库迁移**: `uploaded_by` 列自动迁移，无需手动操作
5. **已有数据**: 已有文档的 `uploaded_by` 为空，对所有用户可见（向后兼容）
