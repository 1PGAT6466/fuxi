## 问题分析

### 1. 认证问题
- 登录接口返回422验证失败，可能原因：
  - 请求格式不符合Pydantic模型验证
  - 密码验证逻辑问题
  - 用户名敏感词过滤

### 2. 速率限制
- 注册接口：每IP每小时最多3次
- 登录接口：每IP每5分钟最多10次
- 测试过程中多次触发限制

### 3. ChromaDB问题
- 向量数据库初始化失败：`KeyError: '_type'`
- 影响依赖向量搜索的功能（RAG搜索等）

## 建议

1. **修复认证流程**：检查登录接口的请求格式验证
2. **调整测试策略**：使用已存在的有效用户进行测试
3. **解决ChromaDB问题**：检查数据库配置和版本兼容性
4. **增加测试覆盖**：在获得有效Token后，重新测试所有需要认证的端点
